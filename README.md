# Offcourse marketing site

## Where things live

| Path | Role |
|------|------|
| `src/` | **Source only** (Nunjucks, Markdown, data). Edit here. |
| `assets/` | CSS and static assets (copied into the build). |
| `docs/` | **Built site** (HTML, sitemap, copied assets). Generated by Eleventy. |

The live site must use the **`docs/`** folder as the web root, not `src/` and not the repo root.

## Update copy and ship it

1. Edit files under `src/` (for example `src/_includes/partials/home-content.njk`).
2. Run **`npm run build`**. That refreshes everything under **`docs/`** (including `docs/index.html`, `docs/creator-program/index.html`, etc.).
3. **Commit and push** both your `src/` changes **and** the matching **`docs/`** changes so any host that serves files straight from Git (without running Node) still shows the new copy.

If you use **Netlify**, **Vercel**, or **Cloudflare Pages** with **Build command** `npm run build` and **Publish directory** `docs`, the platform rebuilds on each push; you should still commit `src/` and `assets/` so the build has source to compile.

## Local preview

**If you double-click HTML on disk:** open **`docs/index.html`** only.  
An old `_site/` folder is not used; opening the wrong file will show outdated copy.

```bash
npm install
npm run build   # refresh docs/ after editing src/
npm run serve   # or use this instead of opening files directly
```

Then open the URL Eleventy prints (files are written to `docs/`).
